Zero-Downtime Deployment on Linux

6 min read • February 2026

At SCOTEC LTD, I learned valuable lessons about deploying enterprise applications on Linux servers while maintaining 99.9% uptime. This post covers the strategies and tools that enable zero-downtime deployments.

Deployment Pipeline

Bash Script
#!/bin/bash
# Zero-downtime deployment script

# Pull latest changes
git fetch origin
git checkout main
git pull origin main

# Install dependencies
npm ci --production

# Run database migrations
npm run migrate

# Build application
npm run build

# Health check before switching
curl -f http://localhost:3000/health || exit 1

# Reload application with PM2 (zero-downtime)
pm2 reload app-name --update-env

# Wait for new instances to be healthy
sleep 5

# Verify deployment
curl -f http://localhost:3000/health

echo "Deployment completed successfully"

Key Practices

  • Use PM2 cluster mode for graceful reloads without dropping requests.
  • Implement health check endpoints to ensure the app is ready before routing traffic.
  • Run database migrations before application restart to prevent schema mismatches.
  • Maintain rollback scripts to revert to the previous stable version instantly.
  • Monitor error rates immediately after deployment using real-time logging.

Lessons Learned

The most important lesson was to always have a rollback plan. Even with comprehensive testing, production environments can surprise you. Automated health checks and gradual rollouts are essential for maintaining system reliability and user trust.
